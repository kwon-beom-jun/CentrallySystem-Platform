# centrally-web-vue3-vite

## 🔐 비밀번호 암호화 방식

### 개요
로그인 시 비밀번호는 **RSA 공개키 암호화**를 통해 전송됩니다. 이를 통해 전송 중 비밀번호가 노출되는 것을 방지합니다.

### 암호화 프로세스

1. **프론트엔드 (로그인 시)**
   - 사용자가 로그인 버튼 클릭
   - `/auth/public-key` API로 RSA 공개키 조회
   - `jsencrypt` 라이브러리를 사용하여 비밀번호를 RSA 암호화
   - 암호화된 비밀번호를 `/auth/login` API로 전송

2. **백엔드 (로그인 처리)**
   - 암호화된 비밀번호 수신
   - RSA 개인키로 복호화
   - BCrypt를 사용하여 저장된 해시와 비교 검증

### 기술 스펙

- **암호화 알고리즘**: RSA
- **키 길이**: 2048비트
- **키 갱신 주기**: 1시간마다 자동 갱신
- **키 저장 위치**: 서버 메모리 (인스턴스당 1개 키 쌍)

### 키 관리 방식

#### 메모리 기반 단일 키 쌍 관리
- 서버 시작 시 RSA 키 쌍 자동 생성
- 모든 사용자가 **동일한 공개키**를 사용
- 키는 **서버 메모리에만** 저장 (디스크 저장 없음)
- 1시간마다 자동으로 새 키 쌍 생성 (기존 키는 폐기)

#### 메모리 사용량 분석

**RSA 2048비트 키 쌍 크기:**
- 공개키 (PublicKey 객체): 약 **294 바이트**
- 개인키 (PrivateKey 객체): 약 **1,217 바이트**
- Base64 인코딩된 공개키 문자열: 약 **392 바이트**
- Base64 인코딩된 개인키 문자열: 약 **1,624 바이트**
- **총 메모리 사용량: 약 2~3KB**

**추가 메모리 오버헤드:**
- KeyPair 객체 래핑: 약 100바이트
- ConcurrentHashMap (키 타임스탬프 추적): 약 1KB
- **총 예상 메모리: 약 4~5KB**

#### 대규모 사용자 처리 가능성

1. **키는 사용자 수와 무관**: 서버에 키 쌍은 **오직 1개만** 존재
   - 사용자 1명이든 100만 명이든 메모리 사용량 동일 (약 4~5KB)
   - 각 사용자가 독립적인 키를 가지지 않음

2. **동시 로그인 처리:**
   - 동일 공개키로 암호화된 비밀번호들을 동일 개인키로 복호화
   - 복호화는 **CPU 연산**이므로 메모리 사용량 증가 없음
   - 동시 1만 명 로그인 시에도 키 메모리는 동일

3. **메모리 효율성:**
   - 서버 인스턴스당 **약 5KB**만 사용
   - 10개 서버 인스턴스 운영 시에도 총 **약 50KB** 정도만 사용
   - 일반 서버 메모리 (2GB~16GB) 기준으로 무시할 수 있는 수준

### 보안 고려사항

#### 장점
✅ **전송 중 비밀번호 보호**: HTTPS와 함께 이중 암호화 제공  
✅ **중간자 공격 방지**: 네트워크 상에서 평문 비밀번호 노출 방지  
✅ **키 자동 갱신**: 1시간마다 새 키 생성으로 보안 강화  
✅ **평문 비밀번호 차단**: 백엔드에서 평문 비밀번호 요청 자동 거부

#### 주의사항
⚠️ **키 갱신 시 잠깐의 로그인 실패 가능**: 키 갱신 중간에 암호화된 비밀번호를 보낸 경우 실패할 수 있음  
  - **해결책**: 키 갱신 실패 시 프론트엔드에서 재시도 로직 구현 권장

⚠️ **서버 재시작 시 키 초기화**: 서버 재시작 시 새 키 생성되므로 이전 키로 암호화된 요청은 실패  
  - **해결책**: 서버 재시작 후 프론트엔드에서 공개키 재조회 필요

### 관련 파일

**프론트엔드:**
- `src/utils/rsaEncryption.js`: RSA 암호화 유틸리티
- `src/api/auth/LoginApi.js`: 공개키 조회 및 로그인 API
- `src/views/UserLogin.vue`: 로그인 화면 (암호화 로직 포함)

**백엔드:**
- `centrally-system-auth/.../RsaPasswordUtil.java`: RSA 암호화/복호화 유틸리티
- `centrally-system-auth/.../RsaKeyService.java`: 키 쌍 관리 서비스
- `centrally-system-auth/.../AuthLoginService.java`: 로그인 서비스 (복호화 로직 포함)
- `centrally-system-auth/.../AuthLoginController.java`: 공개키 제공 API

### 설정 파일

**게이트웨이:**
- `/auth/public-key` 엔드포인트가 JWT 인증 제외 목록에 등록됨
- `centrally-system-config/.../gateway.properties` 파일의 `jwt.excluded-paths` 확인
