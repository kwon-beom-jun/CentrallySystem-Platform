## 📋 프로젝트 개요
**Centrally System**는 영수증 관리, 인사 관리, 정보 공유를 통합한 기업용 중앙 관리 시스템입니다. <br>
영수증 디지털화 및 결재 프로세스 자동화, 직원 인사 정보 및 실적 관리, 공지사항 및 게시판을 통한 내부 커뮤니케이션 등 기업 운영에 필요한 핵심 기능을 제공합니다.
<br><br>


### 🎯 프로젝트 필요성
> 1. **내부 커뮤니케이션 및 정보 공유 체계 구축**
>    - 회사 내부에 별도의 게시판 시스템이 없어 직원 간 정보 공유와 소통이 어려웠음.
>    - 공지사항, 자료실, 자유게시판을 통한 체계적인 정보 공유 시스템 구축으로 업무 효율성 향상.
>    - 부서별, 권한별 가시성 제어를 통한 안전한 정보 관리 및 공유 환경 제공.
>
> 2. **영수증 A4용지 및 수기 작성의 불편함 해소**
>    - 기존에는 A4용지에 영수증을 붙이고 수기로 작성하여 제출해야 했는데, 이 과정이 매우 번거로웠음.
>    - 이를 디지털화하여 효율성을 높이고, 수기 작성에 따른 불편함을 해소하고자 함.
> 
> 3. **영수증 취합 및 제출 과정의 효율화**
>    - 영수증을 수집하여 회사로 직접 전달하는 과정에서 발생하는 불편함과 시간 소모를 줄이고자 함.
>    - 디지털 시스템을 통해 영수증을 빠르고 쉽게 취합 및 제출할 수 있도록 개선하려는 목적.
> 
> 4. **월급 지급 시 영수증 처리의 편리성**
>    - 월급 지급 시 영수증 처리와 관련된 계산 작업을 자동화하여, 기존 수작업으로 인한 오류나 시간 소모를 줄이고자 함.
>    - 특히 월급 지급일 기준으로 영수증 총액을 자동으로 계산해주는 기능을 통해 관리자와 직원 모두에게 편리함을 제공하려는 목적.
> 
> 5. **관리자의 개인별 및 부서별 현황 파악의 용이성**
>    - 관리자가 개인별, 부서별로 얼마나 지출이 발생했는지, 영수증이 어떻게 처리되고 있는지 쉽게 파악할 수 있는 시스템을 제공하여 관리 효율성을 높이기 위함.
>    - 이를 통해 부서별 예산 관리와 비용 절감에 도움을 주려는 목표.
> 
> 6. **관리자의 인적 관리 편리성**
>    - 관리자가 개인의 파견지 실적 등 관리의 편리성을 제공하고자 함.

## 💻 기술 스택

### 백엔드
- **언어**: Java 17
- **프레임워크**: Spring Boot 3.2.x, Spring Cloud 2023.0.0
- **데이터 접근**: Spring Data JPA, QueryDSL, R2DBC
- **보안**: Spring Security, JWT, OAuth2 (Google, Kakao)
- **메시징**: Apache Kafka
- **데이터베이스**: PostgreSQL 16.3

### 프론트엔드
- **프레임워크**: Vue 3 (Composition API)
- **UI 라이브러리**: Vuetify 3, BootstrapVue3
- **상태 관리**: Pinia
- **라우팅**: Vue Router
- **HTTP 클라이언트**: Axios
- **다국어**: Vue I18n
- **빌드 도구**: Vite

### 인프라 & DevOps
- **웹 서버**: Nginx (리버스 프록시 및 정적 파일 서빙)
- **CI/CD**: Jenkins (자동 빌드 및 배포, GitHub 웹훅 연동)
- **형상관리**: Git, GitHub (웹훅 기반 자동 빌드)

<br>


## 🏗️ 시스템 아키텍처

### 마이크로서비스 구성
- **Auth Service**: 인증 및 권한 관리
- **Gateway Service**: API Gateway, 활동 이력 관리, 시스템 로그
- **HRM Service**: 인사 관리 (사용자, 조직, 실적)
- **Receipt Service**: 영수증 관리 및 결재
- **Info Service**: 정보 관리 (공지사항, 자료실, 게시판)
- **Config Service**: 중앙 설정 관리
- **Eureka Service**: 서비스 디스커버리

<br>

## ⚠️ 중요 안내

**현재 프로젝트는 포트폴리오용으로 일부 필수 기능이 비활성화되어 있습니다.**

### 포트폴리오용 비활성화/제거된 기능

- **인증 관련**: 로그인, 회원가입
- **영수증 결재 관련**: 영수증 생성, 제출, 승인/반려, 마감, 일괄 반려, 결재선 제거, 상태 변경
- **정보 관리 관련**: 게시글 생성, 일정 생성
- **데이터 초기화**: 각 서비스의 데이터 초기화 기능
- **설정 데이터(민감정보 포함)**: 모든 application.properties 파일의 실제 비밀번호, JWT 시크릿, OAuth 클라이언트 시크릿, 데이터베이스 연결 정보, SMTP 설정 등은 더미 값으로 전환됨

## ✨ 주요 기능

### 🔐 인증 및 권한 관리
```
- 이메일/비밀번호 기반 회원 가입 및 로그인
- OAuth2 소셜 로그인 (Google, Kakao)
- JWT 기반 인증 및 권한 관리
- 역할 기반 접근 제어 (RBAC)
- 임시 사용자 승인 프로세스
```

### 👥 인사 관리 (HRM)
```
- 사용자 정보 및 프로필 관리
- 조직 구조 관리 (부서/팀)
- 직원 실적 관리 및 이력 조회
- 권한 부여 및 관리
- 즐겨찾기 메뉴 관리
```

### 📄 영수증 관리
```
- 영수증 등록 및 이미지 업로드
- 다단계 결재 프로세스
- 영수증 상태 추적 (대기/승인/반려/확정)
- 월별/연간 영수증 통계 및 분석
- 영수증 카테고리 관리
```

### 📢 정보 관리 (Info)
```
- 공지사항: 공지사항 작성, 조회, 관리
- 자료실: 파일 업로드/다운로드 및 관리
- 자유게시판: 게시글 작성, 수정, 삭제, 댓글 기능
- 게시판 관리: 게시판별 권한 설정 및 가시성 제어
- 사용자 가이드
- 일정 관리: 일정 생성, 수정, 삭제, 조회 (캘린더 뷰)
- 일정 유형 관리: 일정 유형별 분류 및 색상 관리
- 일정 알림: 이메일 기반 일정 알림 발송 (스케줄러 기반 자동 발송)
- 일정 통계: 일정별 통계 및 분석
```

### 📊 시스템 관리
```
- 활동 이력 모니터링
- 통계 대시보드 (부서별/항목별 비용 분석)
- Kafka 메시지 모니터링
- 시스템 로그 관리
```

## 🔒 보안

### 인증 및 인가
- JWT 토큰 기반 인증 (Access Token, Refresh Token)
  - 토큰 만료 시간 관리 및 자동 갱신 메커니즘
- OAuth2를 통한 소셜 로그인 지원
  - Google, Kakao 소셜 인증 연동
- RSA 암호화를 통한 비밀번호 전송
  - 클라이언트 측 비밀번호 암호화 후 전송
- 세션 기반 인증 관리
  - 서버 측 세션을 통한 사용자 상태 관리

### 데이터 보안
- 민감 정보 암호화 저장
  - 비밀번호 등 민감 데이터 해시 암호화 저장
- HTTPS 통신 지원
  - 모든 통신을 HTTPS로 암호화
- CORS 정책 적용
  - 허용된 도메인에서만 API 접근 가능
- SQL Injection 방지
  - JPA를 통한 파라미터 바인딩으로 SQL Injection 공격 방어

### 시스템 보안
- API Gateway를 통한 단일 진입점 관리
  - 모든 요청을 Gateway를 통해 라우팅하여 보안 정책 일원화
- 마이크로서비스 간 통신 보안
  - 서비스 간 인증된 통신 보장
- 역할 기반 접근 제어
  - 사용자 역할에 따른 API 접근 권한 제어
- 활동 이력 추적 및 감사
  - 모든 사용자 활동 로그 기록 및 보관
